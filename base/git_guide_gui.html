What is Git

Git allows you to version control your code. This code will be stored in a git repository. On your device, this will look like a folder than contains your project.

With git, you can:

  • Back up your code
  • Get your code on any device and sync changes
  • Pull in other changes that other people have made to a piece of code
  • Track the history of changes to code - and revert to previous version if needed

Getting Started

Using GitHub requires making an account. Find details on accessing the AMT GitHub group here

This guide uses VS Code. You can download VS Code here. You will also need to download git here.

Starting a Project

In VS Code, open the code you want to use git for. Open the Source Control view on the left side bar (Ctrl+Shift+G) and click Initialise Repository, and Publish to GitHub to put it on GitHub. You may need to connect your GitHub account for this.

You can access remote repositories on your device by opening the Source Control view on the left side bar (Ctrl+Shift+G) in an empty VS Code window, and click Clone Repository. This will allow you to input the URL of the repository you wish to active.

Making Changes to Code

Before you make any changes, make sure your local code is up to date with the remote repository. Open the Source Control view on the left side bar (Ctrl+Shift+G), click the ... button on the REPOSITORIES section, and click Pull.

You can make changes to the code in the local repository on your machine in the same way you would edit any other code. You can check which files you have made changes to by opening the Source Control view on the left side bar (Ctrl+Shift+G). You will need to stage and commit these files to save your changes to your remote repository.

To stage your files, hover over each file in the list of changed files and click the + button. If you accidentally stage a file you did not want to stage, you can unstage it by hovering over it and clicking the - button.

After you have added all your files, you need to add a message explaining the changes you have made. Enter your commit message in the message box above the list of changed files and click the commit button to commit your changes.

These changes are currently only available to you locally. You can apply these changes to the remote repository by opening the Source Control view on the left side bar (Ctrl+Shift+G) and clicking the sync changes button. Afterwards, your changes should be available in the remote repository.

Working With Others: Branches

Branches

If there is more than one person working in a repository, it is common that each person works on their own branch. This means, if you were to make and push changes, the other person will not get the changes you have made unless they choose to merge your changes into their own branch. This means you can safetly push changes without conflicting with the other person's work.

Typically, there will be a main branch from which the other branches will stem from. Once you are satisfied with the changes you have made to your branch, you can merge your changes to the main branch. Everyone else working on the repository will then be able to get these changes from the main branch, and merge them into their own.

Branch Commands

You can check what branch you are on in Source Control view, or on the status bar at the

To switch to another branch, select the branch you want to move to from the status bar.

To create a new branch, click your current branch name on the bottom status bar and enter the name of your new branch. Click the branch you want to base your new branch from, usually main, and VS Code will switch to your new branch. This should appear on your remote repository when you next sync your repositories.

If you need to merge changes from your new branch back to the source branch, switch to the branch you want to merge to. Then you can open the Command Palette (Shift+cmd+P), type Git: Merge branch and select the branch you want to merge from.

Additional Settings

SSH Keys

This section requires CLI use. It is necessary to allow you to easily connect with remote repositories.

You may need to connect to GitHub repositories with SSH. In this case, you will need to set up an SSH key to connect with.

First, you should check if you have any existing SSH keys. You can do this by running the following commands to list any files in your .ssh directory:

ls -al ~/.ssh

Windows ls -al ~/.ssh
Mac ls -al ~/.ssh

If you need to generate generate an SSH key, run the following command and replace your_email@example.com with the email associated with your GitHub account:

ssh-keygen -t ed25519 -C "your_email@example.com"

Windows ssh-keygen -t ed25519 -C "your_email@example.com"
Mac ssh-keygen -t ed25519 -C "your_email@example.com"

You will be prompted with Enter file in which to save the key, for which you can press enter to accept the default location. You will also be prompted choose a passphrase. Press enter if you do not want a passphrase.

You must now add your pubic key to your GitHub account. Copy the contents of the .pub file generated. With a public file named id_ed25519.pub, you could do this with the following command:

cat ~/.ssh/id_ed25519.pub

Windows clip < ~/.ssh/id_ed25519.pub
Mac pbcopy < ~/.ssh/id_ed25519.pub

Now, on your GitHub account, go to Settings > SSH and GPG keys and click New SSH key. Paste the .pub file contents in the Key box, give it a title, and save. You should now be able to connect to repositories via SSH.
